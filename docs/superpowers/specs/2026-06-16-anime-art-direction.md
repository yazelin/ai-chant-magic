# 真。AI。咏唱魔法 — 美術方向重訂(anime + 浮誇特效)

- **日期**:2026-06-16
- **背景**:第一版美術(AI 暗黑肖像 + 把肖像硬旋轉)被 yazelin 評為「醜 + 機械、不像魔法」。本文件記錄重訂後的方向。

## 為什麼醜(診斷)

不是「正上俯視」的錯(Brotato/Hades/Gungeon 都是俯視且好看),是 **風格(AI 暗黑寫實肖像)+ 旋轉肖像** 的錯。修正:換 anime 風 + 角色不旋轉 + 法術特效撐魔法感。

## 鎖定的決定

1. **視角**:維持正上俯視(不改成 LF2 側捲)。
2. **移動 / 面向**:移動完全自由(WSAD,LF2 也能上下走);角色**只左右翻轉(flip L/R)、不旋轉**;法術**朝滑鼠 360°** 飛(身體 flip 只看 aim 的左右)。這組合避開「肖像旋轉躺倒」的醜。
3. **角色風格**:**anime 賽璐珞遊戲角色**(KonoSuba archetype 神似不照搬:火法師少女 / 冰 / 雷 / 守護),用 **codex-imagegen 生成**。
4. **動畫做法(關鍵)**:codex 做不出一致逐格動畫,所以用 **pose-transfer 混合法**——
   - 拿現成動畫(calciumtrice「Animated Wizard」,**CC-BY 3.0**,README 標註作者)的**姿勢幀當骨架**;
   - 我們的 anime 法師當**風格參考**;
   - codex edit 模式(多參考圖)逐幀重畫成「我們風格 + 那個姿勢」。**已驗證**:站立↔施法兩幀角色/風格一致。
   - 幀數先精簡(待機 + 施法),其餘靠**程序動畫**(走路上下彈、施法縮放閃光、朝滑鼠微傾)。
5. **法術特效**:**超浮誇**是魔法感主力——發光核心 + additive 光暈 + 拖尾粒子 + 命中爆光 + 螢幕閃/震。
6. **怪物**:codex 生 anime 風怪物(1-2 種)。
7. **授權**:pose 骨架 calciumtrice CC-BY(標註);codex 輸出為原創 anime 風(yazelin ChatGPT 額度,個人用)。素材進公開 repo 前確認授權乾淨。

## 製作策略:pilot 先行

先把 **火法師 pyro 一條龍**做完進遊戲(anime 待機/施法幀 + flip + 程序動畫 + 火球浮誇特效)→ 部署給 yazelin 看。**確認遊戲裡好看,才照同 pipeline 量產**其餘 3 職業 + 怪物。避免一次生一堆才發現不對。

## 驗收(pilot)

線上(或本機)看 pyro:站著有呼吸感、移動會翻面/彈跳、施法切到施法姿勢 + 火球是大團發光帶拖尾、命中有爆光。yazelin 點頭 → 量產。
